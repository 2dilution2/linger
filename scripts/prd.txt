# Overview  
Linger(시가 머무는 순간)는 감정이 북받칠 때 누구나 필명으로 시를 쓰고, 감성적인 배경과 폰트로 자신만의 감정을 표현하며, 익명으로 소통할 수 있는 감성 커뮤니티 플랫폼입니다. 단순한 시 공유를 넘어, 감정이 머무는 찰나를 기록하고, 공감과 위로를 주고받는 공간을 지향합니다. 특히 새벽 시간대, 감성적인 순간에 머무는 20~30대 여성 유저를 주요 타겟으로 합니다.

# Core Features  
- 필명 기반 간편 회원가입(카카오/구글)
- 커스터마이징 가능한 배경 테마(사진, 그라데이션, 직접 업로드)
- 다양한 감성 폰트 제공(고딕, 손글씨, 붓글씨 등)
- 감정 태그 시스템 (#기쁨 #그리움 #불안 #사랑)
- "구(句)" 기능: 한 줄 시 작성, 실시간 구 공유, 구 모음집, 감정 분석
- 시를 이미지로 저장하여 SNS 공유(추가)
- 월간 달력 UI로 시 작성 기록 관리(후순위)
- 감정 기반 색상/아이콘 표시(후순위)
- 커뮤니티: 감정/테마별 피드, 인기 시 큐레이션, 댓글/공감/즐겨찾기/핀/구독
- 감정 분석: 월간 감정 태그 통계, 단어 분석, 감정 변화 추적, 회고(후순위)
- AI 보조 기능: 시 작성 보조, 감정 기반 추천, 유사 감정 사용자 매칭, AI 자동 시 업로드(초기 유입용)

# User Experience  
- 주요 페르소나: 20~30대 여성, 감성적, 새벽 시간대에 감정 공유를 원하는 사용자
- 첫 진입 플로우: 카카오/구글 간편 회원가입 → 필명 입력 → "시를 쓸래요"/"볼래요" 선택 → 시 작성 또는 커뮤니티 피드 진입
- UI/UX 원칙: 다크모드 기본, 라이트모드 선택, 감성적이고 직관적인 인터페이스, 최소한의 클릭으로 감정 표현 가능

# Technical Architecture  
- 프론트엔드: React Native, TypeScript, Styled-components, Redux Toolkit
- 백엔드: Node.js, Express, MongoDB, JWT
- 공통: 클라이언트/서버 공용 타입 및 상수(shared)
- 에러 관리: Sentry(후순위)
- 배포: 초기에는 로컬 서버, 추후 사용자 증가 시 클라우드 이전 고려

# Development Roadmap  
- 1차(MVP): 시 작성, 감정 태그, 배경/폰트 커스터마이징, 구 기능, 커뮤니티(피드/댓글/공감/핀/구독), 간편 회원가입, 다크모드, AI 보조(시 작성)
- 2차(후순위): 달력 기반 기록, 감정 분석, 감정 변화 추적, 시 이미지 저장 및 SNS 공유, Sentry 연동, 감성 메시지, 회고 기능

# Logical Dependency Chain
1. 시 작성(에디터, 태그, 커스터마이징, 구)
2. 커뮤니티(피드, 댓글, 공감, 구독 등)
3. 감정 분석(통계, 변화 추적)
4. 달력 기반 기록 및 감성 메시지

# Risks and Mitigations  
- 사용자 유입 저조: 초기에는 AI가 자동으로 시를 업로드하여 피드 활성화
- 감정 분석/추천 품질: AI 모델 지속 개선 및 피드백 반영
- 서버 비용: 초기에는 로컬 서버, 트래픽 증가 시 클라우드로 확장
- 기술적 리스크: Sentry 등으로 에러 모니터링, 빠른 대응 체계 구축
